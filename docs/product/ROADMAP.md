# Актуальный roadmap продукта

Обновлено: 12 июня 2026.

Этот файл содержит только актуальные задачи. Уже реализованные идеи вынесены из backlog, чтобы не создавать ложное ощущение незавершённой работы.

## Текущее состояние

Бот уже пригоден для использования небольшой группой людей:

- публичная автоматическая регистрация пользователей;
- роли `member`, `admin`, `owner` и owner-only backup;
- welcome-first onboarding, пресеты и ручная настройка профиля;
- profile health, мягкий gate недельной выдачи и диагностика;
- персональные фильтры, стоп-слова и язык вакансий;
- уведомления, недельная выдача и ручная пересборка;
- статусы `Сохранить`, `Откликнулся`, `Скрыть`;
- фильтр резюме и объяснение причин совпадения;
- exact/canonical dedup и ссылки на посты-дубли;
- Telegram web polling с cursor и inactivity backoff;
- управление каналами, пакетное добавление и бессессионный channel discovery через mention graph/manual batches; MTProto остаётся optional provider;
- экспериментальные источники hh.ru и сайты компаний;
- локальная аналитика в SQLite и optional PostHog;
- admin alerts, backup, security-hardening и runtime settings;
- единая UX-геометрия кнопок и подтверждение сброса профиля.
- bot-layer разделён на orchestration, panels, onboarding, input flows и keyboards; `createBot.ts` снова проверяется строгим TypeScript.
- автоматический smoke запуска, Telegram polling, heartbeat и healthcheck проходит; startup diagnostic отправляется до долгого первого poll-цикла.
- тяжёлые ручные действия защищены cooldown: пересборка пользователя ограничена на пользователя, discovery и пакетное добавление каналов — глобально, backup — один на все ручные пути, проверки career-site — отдельно на источник.
- техническая история автоматически очищается по retention без удаления вакансий, пользовательских данных и blocked/pending discovery-кандидатов.
- одиночные уведомления используют компактные структурированные карточки с раскрытием полного текста.
- вакансии можно отложить: напоминания хранятся в SQLite, переживают рестарт и повторяют доставку после ошибки.
- каждый поиск показывает фактическую отдачу за 7 дней, а пресеты показывают локальный прогноз совпадений до применения.
- weekly zero-state объясняет отсутствие выдачи через причины отсева, скрытые совпадения или отсутствие накопленных данных.
- структурированные Telegram-посты с несколькими вакансиями разделяются на отдельные карточки; старые агрегаты можно безопасно repair'ить.
- доверенные внешние job-сервисы управляются через admin UI и безопасно обогащают карточки только по exact-host allowlist.
- локальная Windows-эксплуатация без Docker поддерживает автозапуск, перезапуск, healthcheck и автоматические retained backup.

## Рекомендуемый порядок

### Следующий короткий цикл

1. Провести ручной smoke последних UX-изменений.

### После нескольких недель использования

1. Сделать автоматический owner-report по аналитике.
2. Посмотреть реальные причины шума и пропусков.
3. Добавить relevance feedback.
4. Решить, нужен ли fuzzy dedup и какие каналы стоит отключить.
5. Проверить долгосрочный рост вакансий, raw messages и пользовательских данных.

### Не делать пока

- платежи и тарифы;
- полноценную платформу вместо Telegram-бота;
- хранение резюме;
- сложный AI matching;
- автоматическое добавление найденных каналов без owner review.

## Критерий готовности для текущего небольшого запуска

Можно считать бот завершённым для текущего кейса, когда:

- он автоматически запускается и восстанавливается локально;
- owner получает startup/failure alerts и регулярный backup;
- живой smoke проходит без ошибок;
- публичный пользователь не может спамить тяжёлыми операциями;
- несколько недель реального использования не показывают критичных проблем качества выдачи.

Связанные открытые решения находятся в [QUESTIONS_AND_IDEAS.md](./QUESTIONS_AND_IDEAS.md).
