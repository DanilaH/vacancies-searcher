# Product Docs

This folder stores product-facing planning artifacts for the bot.

Read order:
1. [PROJECT_OVERVIEW.md](./PROJECT_OVERVIEW.md)
2. [ROADMAP.md](./ROADMAP.md)
3. [QUESTIONS_AND_IDEAS.md](./QUESTIONS_AND_IDEAS.md)

Use these docs for:
- prioritizing feature work;
- keeping product ideas in one place;
- turning rough notes into implementation tasks later;
- helping a fresh agent understand what matters now versus later.

`PROJECT_OVERVIEW.md` is the canonical current-state description for research and handoff.

`ROADMAP.md` contains only unfinished product and functional priorities.

If product docs and technical docs overlap:
- product intent lives here;
- code ownership and dependency routing live in [../agent-map/README.md](../agent-map/README.md).
